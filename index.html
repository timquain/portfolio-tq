Artifact Information: JayTees Fabric Assistant (AI E‑commerce Chatbot)

Artifact Title

JayTees Fabric Assistant — AI‑Powered E‑Commerce Chatbot

Introduction

This artifact is a working prototype of an AI shopping assistant for a boutique fabric business. It helps customers discover fabrics, estimate yardage, read care instructions, and understand policies through natural conversation. The prototype was developed during an AI Lab exercise.

Artifact Description

Objective: Design and implement a helpful, safe, and testable AI assistant that improves customer decision‑making and reduces repetitive support.

Process: Empathize with users → Define the problem → Ideate intents and flows → Prototype (FastAPI + OpenAI) → Test (smoke, scenarios, unit tests) → Iterate (prompt tuning, cross‑sell rules, retrieval quality).

Was it a group or individually created product? Individually created; collaboration‑ready architecture with clear runbook and testing utilities.

Tools and Technologies Used: Python, FastAPI, Uvicorn, OpenAI API, YAML scenario tests, unittest, JSON product catalog, HTML/JS chat UI, runtime config and feedback logging.

Artifact‑Specific Value Proposition

Unique Value: Demonstrates end‑to‑end product thinking: user empathy, conversational design, safety guardrails, measurable testing, and deploy‑ready backend.

Relevance: Aligned to my personal value proposition by showcasing both UX clarity and technical execution; relevant to audiences evaluating practical AI skills for real‑world applications.

Customization for Audience

Adaptations Made: Branded as “JayTees Fabric Assistant,” added concise tone and clarifying questions, product cards, and optional visual UI. Provided a verification runbook and a portfolio page for academic or hiring review.

Relevance: Customizations ensure the artifact is accessible to non‑technical reviewers while retaining technical depth for developers.

Reflection

Significance: I chose this artifact because it represents the fusion of AI capability with human‑centered design. It demonstrates practical skill across prompts, back‑end services, and evaluation tooling.

Lessons Learned: I learned how to structure AI prompts around user intents, tune response behavior with runtime config, and validate outcomes with automated tests. I also reinforced the importance of guardrails, concise UX copy, and iterative refinement.

Feedback and Revisions

Feedback: Early feedback emphasized clarity of instructions and a desire for a visual demo. I incorporated a runbook, smoke tests, and a branded UI concept with persistent history.

Revisions: Improved onboarding (README and runbook), better error handling in test scripts, added /config and /feedback endpoints, and refined prompts for specific intents (yardage, care, policies).

References

FastAPI documentation; OpenAI API reference; internal runbook and testing scripts developed during the AI Lab.
